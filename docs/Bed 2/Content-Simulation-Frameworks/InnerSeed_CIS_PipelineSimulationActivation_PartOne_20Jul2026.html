Activation

The three milestones

M1 — First real run. The pipeline executes end-to-end on the desktop and stops, by design, at READY FOR APPROVAL — a real Higgsfield clip URL in the pending record. That stop is success: Claude's finish line is ready-for-approval, never published.

M2 — The archive path, for real. Same pipeline, archived input: the March Reel caption goes through Insight, Quality holds on the stale dated line, returns the fix, passes on re-check. The simulation's second path, no longer simulated.

M3 — The gate goes live. approve actually schedules through Postiz. Blocked until the Postiz self-hosting decision — not wired before then, on purpose.

Before starting

The human prerequisites

1 · Higgsfield account and API key. Does not exist yet — the one true blocker for M1, and the first real spend decision on the visual seat. Created at cloud.higgsfield.ai, key noted, pricing terms read.

2 · Anthropic API key. The script's internal quality pass calls the API directly — separate from the Claude Code login.

3 · The D-43 screen recording, saved locally as d43_reference.mp4 in the project folder. For M1 a local file replaces the Drive fetch entirely — the simplest honest path.

4 · Postiz — not yet. Every POSTIZ_* variable stays unset for M1 and M2. The script structurally cannot publish without them, which is exactly right.

Desktop setup

Running it through Claude Code

One project folder — ~/innerseed-pipeline — holding the runnable counterpart to this page, InnerSeed_CIS_Level2_PipelineReference_19Jul2026.py, the two master documents for context, and the local reference recording. git init, keys in a .env that never gets committed, then claude in that folder and this first instruction:

Read InnerSeed_CIS_Level2_PipelineReference_19Jul2026.py and skim both HTML masters for context. Before running anything: (1) verify the Higgsfield endpoint shapes in the script against Higgsfield's current developer docs and correct any drift; (2) replace the drive_download step with reading the local file d43_reference.mp4; (3) load keys from .env. Then run the new-footage path: python InnerSeed_CIS_Level2_PipelineReference_19Jul2026.py run --input new Stop at READY FOR APPROVAL. Do not install, configure, or call Postiz. Show me the pending record and the clip URL when done.

M1 success check: a file appears in pending_approvals/, the terminal reads READY FOR APPROVAL, and the clip URL opens and plays. Watching that clip is the human gate doing its job — approve stays unrun.

House rules

Unchanged at every level of automation

Every pass ends pass-or-hold. A hold always returns with instructions, never silence. Old content gets the same Insight pass as new. Publishing requires an approval record to exist — a missing approval means a stopped pipeline, not a published mistake. And the quality bar includes reading as human-authored, every time.
