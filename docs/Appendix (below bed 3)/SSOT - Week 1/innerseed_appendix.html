01 — Agreed Project Framework (Fixed)

The following was agreed on 8 June 2026 and is the structure we work to from this point forward. It does not change without a logged decision.

F.01Document Structure — Final Agreed Framework

Four documents total. No more, no less unless a change is logged.

  • Doc 1 — InnerSeed Development & Strategy Log — Weekly session notes + strategy + this appendix registry. One living file. Primary working document for ISD and source material for university assignments.
  • Doc 2 — Brand Book & Marketing Tool — Standalone professional asset. E-portfolio piece.
  • Doc 3 — Reflective Report + CPD Plan — University Assignment 2. 3,000 words. Formal academic submission.
  • Doc 4 — E-Portfolio — University Assignment 1. Built in Canva or Wix. Draws artefacts from Docs 1 and 2. Includes compulsory 500-word placement blog.
F.02Project Split — Professional vs University Framework

70% professional working relationship with ISD. 30% university module I4168. The two are complementary, not the same. Documents serve ISD first and university second — not the other way around.

F.03Project Vision — InnerSeed Garden Framework

InnerSeed Garden is a mindfulness and guided session app. Core purpose: empowering users to integrate positive change into daily life and find present moment awareness. Key theoretical and commercial backbone: Michael Brown's The Presence Process. More sources to be added over time and logged in Section 05 of this appendix.

F.04Appendix Role — Confirmed Framework

The appendix records everything: documents created, links, decisions, frameworks, sources, session summaries, chat logs. It lives inside Doc 1 as a section and is also saved as a standalone file (this file). It is the memory cloud for the project.

3,000 words (+/- 10%). Must use a named reflective framework (Kolb, Gibbs, Rolfe etc). Three sections: Introduction (~200w), Reflection on learning (~1,500w), Future development / CPD action plan (~1,300w). Harvard referencing. Word document submitted to Canvas.

D.03Week 2 — UX Research Brief Dev Log

Week 2 focus is regular user experience testing across core app flows. The guiding development philosophy throughout is: keep it simple, supergastric-case functionality — i.e. design for the most natural, lowest-friction path for the everyday user.

Areas to explore and document this week:

  • Preset mode — how does it feel as a regular user?
  • Filter mode — is it intuitive and discoverable?
  • Saving sessions — is the process clear and easy?
  • Creating sessions — what is the experience of building from scratch?
  • Favouriting sessions — does this feel natural?
  • General — anything that feels unnatural, complex or unnecessarily effortful
