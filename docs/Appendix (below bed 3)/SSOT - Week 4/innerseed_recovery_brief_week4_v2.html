Recovery Brief — Week 4

Week: 4 Dates: 22–24 Jun 2026 Author: Ruaraidh Scope: Week 4 only

Scope of this document: This recovery brief covers Week 4 only (22–24 June 2026). For the full project context — people, background, document structure, university deadlines and all prior weeks — refer to the master recovery brief (innerseed_recovery_brief.html).

01 — Week 4 in one paragraph

Week 4 completed the full desktop audit of Builder mode — the app's most commercially significant feature — using the Builder Mode Evaluation Framework agreed at the start of the week. Eleven findings were logged (D-39 to D-49). The week's standout discovery was a Spotify playlist integration hidden behind an unlabelled icon: a feature that allows users to bring their own Spotify playlists into a breathwork session, with no equivalent in mainstream wellness competitors. The transport bar was fully decoded — its five unlabelled icons were confirmed to perform filter access, mode switching, and Share/Save, all invisible without accidental discovery. A full-width stacked Builder layout and a completed session share card were produced as visual directions for application development. The Week 4 Manifesto contains 16 embedded figures drawn from raw testing screenshots and session concepts. Running findings total: D-01 to D-49.

02 — What was tested this week

Builder mode — Ambient Music pillar: Track selector, loop control, volume slider, and Spotify integration. Spotify found to be hidden behind an unlabelled playlist icon — a curated "Garden Playlist" plus a field to paste a personal playlist URL. No mainstream competitor currently offers equivalent user playlist import functionality. Proposed fix: surface Spotify as a named, visible feature within the expanded card.

Builder mode — Guided Breaths pillar: Dropdown selector ("Diepe Neusademhaling" in Dutch — D-41, fourth confirmed context for language instability), volume slider. "Find your breath" modal confirmed as well-designed — search, filter tabs (Balanced/Stronger), descriptions, preview play — but buried behind an unlabelled icon.

Builder mode — Guided Voices pillar: "Select voice…" dropdown produces no options — no voices currently available or displayable (D-49). Volume toggle works correctly. Audit of this pillar cannot be completed until voices are available.

Transport bar — all five icons decoded:

  • Library icon → opens Filter mode; all six filter labels in Dutch (D-45, D-46)
  • Wind/flow icon → switches to Session/Preset mode, unlabelled (D-47)
  • Mixer/sliders icon → switches back to Builder mode, unlabelled (D-47)
  • Stop icon → stops the session
  • Three-dot menu → Share session and Save Session, both hidden (D-48)

04 — Proposals produced this week

Full-width stacked Builder layout (Fig. 14–15 in Manifesto): Three pillars stacked full width instead of compressed into three columns. Numbered steps (1, 2, 3), each pillar with four clearly labelled function columns. Spotify sits as a named, visible fourth column in Step 1. Session summary with Save and Play at the bottom. Developed as a visual direction for application development purposes.

Completed session share card (Fig. 16 in Manifesto): A finished session presented as a shareable product — artwork banner, session title, all three components summarised, Spotify status, favourites status, Share and Save both visible and labelled in the top right. Addresses D-48 and the broader point that a built session should feel like something worth sharing.
