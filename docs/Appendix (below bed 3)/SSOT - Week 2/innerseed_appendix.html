01 — Agreed Project Framework (Fixed)

The following was agreed on 8 June 2026 and is the structure we work to from this point forward. It does not change without a logged decision.

F.01Document Structure — Final Agreed Framework

Four documents total. No more, no less unless a change is logged.

  • Doc 1 — UX Development Manifesto (internal nickname: SSOT) — Weekly UX notes, observations, screenshots, findings and strategy. One living document per week. Primary working document for ISD and source material for university assignments. Renamed 12 Jun 2026, see SL.03.
  • Doc 2 — Brand Book & Marketing Tool — Standalone professional asset. E-portfolio piece.
  • Doc 3 — Reflective Report + CPD Plan — University Assignment 2. 3,000 words. Formal academic submission.
  • Doc 4 — E-Portfolio — University Assignment 1. Built in Canva or Wix. Draws artefacts from Docs 1 and 2. Includes compulsory 500-word placement blog. Status: in motion — evidenced as Doc 1 is built week by week.
F.02Project Split — Professional vs University Framework

70% professional working relationship with ISD. 30% university module I4168. The two are complementary, not the same. Documents serve ISD first and university second — not the other way around.

F.03Project Vision — InnerSeed Garden Framework

InnerSeed Garden is a mindfulness and guided session app. Core purpose: empowering users to integrate positive change into daily life and find present moment awareness. Key theoretical and commercial backbone: Michael Brown's The Presence Process. More sources to be added over time and logged in Section 05 of this appendix.

F.04Appendix Role — Confirmed Framework

The appendix records everything: documents created, links, decisions, frameworks, sources, session summaries, chat logs. It is the project's master background reference — separate from the UX Development Manifesto (Doc 1), which holds the weekly UX findings, screenshots and design work. The appendix is the memory cloud for the project; the Manifesto is the polished weekly output.

D.05Week 2 — UX Research Brief Dev Log

Week 2 focus is regular user experience testing across core app flows. The guiding development philosophy throughout is: keep it simple, supergastric-case functionality — i.e. design for the most natural, lowest-friction path for the everyday user.

Areas to explore and document this week:

  • Preset mode — how does it feel as a regular user?
  • Filter mode — is it intuitive and discoverable?
  • Saving sessions — is the process clear and easy?
  • Creating sessions — what is the experience of building from scratch?
  • Favouriting sessions — does this feel natural?
  • General — anything that feels unnatural, complex or unnecessarily effortful

SL.04Working Session — 12 June 2026 (continued) Session Log

Completed the opening screen five-question audit (Q3 Trust & tone, Q4 Friction, Q5 Return user consideration). Then began Step 1 of the reorganised testing journey — live Preset playback testing using "Ethereal Voyager."

Q3 (Trust & tone): dashboard reads as sleek/intelligent/“game-like” rather than soft and calming — a tonal mismatch for a wellness product (D-06). Hero banner image cuts off abruptly with a hard edge (D-07).

Q4 (Friction): no single obvious path to "experience something" — the dominant CTA pushes toward building, not listening (D-08); left-hand nav labels (Preparation, Session Builder, The Green House) all describe process/stats, not experience (D-09).

Q5 (Return user): layout still feels overly complex for a simple core task; should be usable "almost like a three year old could use it" (D-10).

Preset playback testing (live): attempting to open a Preset led into Builder mode instead, reproducing W1-01 live (D-11). Session cards only clickable via a tiny play icon, not the whole card (D-12). No way back from a Preset player to the dashboard — nav doesn't offer a route home, and the split InnerSeed logo is ambiguous as a home button (D-13, D-14). The player's control panel doesn't visually read as a breathwork session — reinforces W1-05 breathing visualiser idea (D-15). Session metadata line ("21 min · CONTACT · Connect · ♂ Jeroen") is unclear and may be a labelling bug — flagged for developer check (D-16). Transport bar icons (folder, settings, save) appear to be reused from Builder mode and don't make sense in a Preset playback context (D-17).

Overall: findings now total D-01 to D-17. A recurring pattern has emerged of Builder-mode UI and language bleeding into the Preset/playback experience, compounding the "discovery first" issue identified earlier in the week.
